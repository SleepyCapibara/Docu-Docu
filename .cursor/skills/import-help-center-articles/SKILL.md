# Import Help Center Articles

Imports articles from [SOC Prime Help Center](https://help.socprime.com) into this Docusaurus site using `scripts/import-help-articles.js`.

## When to use

- User asks to import, migrate, or copy help center articles
- User provides a `help.socprime.com/en/articles/...` URL
- User wants to batch-import a docs section from the help center

## Quick workflow

1. Read `scripts/help-center-imports.json` for existing imports
2. Add a manifest entry (or use `--url` for a one-off import)
3. Run the import script
4. Verify build output and spot-check images in the generated MDX

```bash
# Import everything in the manifest
node scripts/import-help-articles.js

# Import one article by slug
node scripts/import-help-articles.js --slug search

# One-off import without editing the manifest
node scripts/import-help-articles.js \
  --url "https://help.socprime.com/en/articles/5820372-dashboard" \
  --slug dashboard \
  --section platform-guides

# Validate fetch/parse without writing files
node scripts/import-help-articles.js --slug search --dry-run
```

## Manifest entry template

Add to `scripts/help-center-imports.json` under `articles`:

```json
{
  "url": "https://help.socprime.com/en/articles/<id>-<slug>",
  "slug": "article-file-name",
  "section": "docs-subfolder-under-product",
  "sidebarPosition": 3,
  "startMarker": "First ~80 characters of the article intro paragraph",
  "subtitle": "Optional subtitle line under the title",
  "author": "Author name from help center",
  "updated": "Updated this week"
}
```

### Field rules

| Field | Required | Notes |
|-------|----------|-------|
| `url` | Yes | Full help center article URL |
| `slug` | Yes | MDX filename (`{slug}.mdx`) and image folder name |
| `section` | Yes | Folder under `docs/product/` (e.g. `threat-detection-marketplace`) |
| `sidebarPosition` | Recommended | Controls sidebar order within the section |
| `startMarker` | Recommended | Unique opening text; script auto-detects if omitted but explicit is safer |
| `title` | Optional | Auto-extracted from page JSON-LD if omitted |
| `subtitle`, `author`, `updated` | Optional | Preserved in MDX front area when provided |

## Output paths

For each article, the script writes:

- MDX: `docs/product/{section}/{slug}.mdx`
- Images: `static/img/docs/{section}/{slug}/01.png`, `02.png`, ...
- Image refs in MDX: `/img/docs/{section}/{slug}/NN.png`

Sidebar is autogenerated from `sidebars.js` — no sidebar edit needed when files land under `docs/product/`.

## New doc section checklist

When `section` does not exist yet:

- [ ] Create `docs/product/{section}/` (script creates on import)
- [ ] Add `docs/product/{section}/_category_.json` if missing, or set `sectionConfig` in manifest:

```json
{
  "sections": {
    "platform-guides": {
      "label": "Platform Guides",
      "position": 4,
      "description": "Core platform documentation."
    }
  }
}
```

Pass `sectionConfig` on the article entry as `sectionConfig` to auto-create `_category_.json` on first import.

## After import

1. Run `npm run build` (script runs this by default; use `--skip-build` to skip)
2. Open the generated MDX and confirm:
   - Intro paragraph present
   - Headings (`##`, `###`) render correctly
   - Screenshots appear in the right sections
   - Lists and notes are readable
3. Append the new entry to `scripts/help-center-imports.json` if imported via `--url`

## Troubleshooting

| Problem | Fix |
|---------|-----|
| `startMarker not found` | Copy the exact opening sentence from the live article into the manifest |
| Missing images | Re-run import; existing files are kept, missing ones are downloaded |
| Broken lists/headings | Help center HTML changed — inspect fetched HTML and update parser in `scripts/import-help-articles.js` |
| Wrong section in sidebar | Adjust `sidebarPosition` in manifest and re-import |

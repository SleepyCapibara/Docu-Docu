У Search ви можете досліджувати Threat Detection Marketplace (TDM) — найбільшу у світі колекцію правил детекції та hunting-запитів.

Кожен елемент контенту спочатку написаний у Sigma — спільній мові кіберзахисту — а потім автоматично перекладається у нативні формати різних SIEM, EDR і XDR. Правила Sigma зіставлені з MITRE ATT&CK® і збагачені розвідданими про пов’язані загрози.

Правила детекції ранжуються та відображаються як списки з ключовими деталями. Ви можете переглядати, шукати та фільтрувати правила Sigma.

Щоб забезпечити точні інсайти в [Dashboard](https://help.socprime.com/en/articles/5820372-dashboard), [Log Source Coverage](https://help.socprime.com/en/articles/5820378-log-source-coverage) та [MITRE ATT&CK Coverage](https://help.socprime.com/en/articles/5820381-mitre-att-ck-coverage) вашої організації, позначайте правила як deployed, якщо ви завантажуєте та встановлюєте їх вручну. Щоб позначити правило як deployed, перейдіть на вкладку Detection Code і натисніть значок галочки в меню дій праворуч. Докладніше див. у цьому розділі.

Щоб автоматизувати пошук, налаштування та розгортання правил, налаштуйте такі функції та модулі:

- [Search Profile](https://help.socprime.com/en/articles/6261210-search-profiles)
- [Custom Field Mapping](https://help.socprime.com/en/articles/6261215-custom-field-mapping)
- [Data Planes](https://help.socprime.com/en/articles/6261097-data-planes)
- [Automation](https://help.socprime.com/en/articles/6261236-automation-getting-started)
- [API](https://help.socprime.com/en/articles/6265791-api)

Щоб перейти на сторінку Search, відкрийте Threat Detection Marketplace і виберіть вкладку Search.

## Search Bar

---

Рядок пошуку розташований у верхньому лівому куті сторінки. Ви можете вибрати один із трьох режимів пошуку: Standard, Lucene і Light.

### Standard

Цей режим увімкнено за замовчуванням. Виконуйте стандартний повнотекстовий пошук по всьому контенту в Threat Detection Marketplace.

![Знімок екрана продукту](/img/docs/threat-detection-marketplace/search/02.png)

Коли ви починаєте вводити запит, з’являються підказки. Виберіть одну з них або завершіть введення та натисніть значок лупи або Enter (Windows)/Return (Mac).

![Знімок екрана продукту](/img/docs/threat-detection-marketplace/search/03.png)

Пропоновані результати можуть базуватися на збігах у таких атрибутах:

- Configs for alternative translations
- Log source products
- Event IDs
- Sigma type

Search показує основну статистику за поточними результатами пошуку.

![Знімок екрана продукту](/img/docs/threat-detection-marketplace/search/04.png)

### Lucene

Щоб увімкнути цей режим, натисніть стрілку вгору поруч із Standard і виберіть опцію Lucene у випадаючому списку.

![Знімок екрана продукту](/img/docs/threat-detection-marketplace/search/05.png)

Використовуйте синтаксис Lucene для складання складних запитів: фільтруйте контент, поєднуйте пошукові терміни з різними логічними операторами та багато іншого. Щоб дізнатися більше про використання Lucene, натисніть значок знака питання праворуч від мітки Lucene. Щоб переглянути список усіх доступних полів разом із їх типами та можливими значеннями, перейдіть до розділу [Lucene](https://help.socprime.com/en/articles/6265772-lucene-syntax) цього посібника.

### Light Search

Щоб увімкнути цей режим, натисніть стрілку вгору поруч із Standard і виберіть опцію Light Search у випадаючому списку.

![Знімок екрана продукту](/img/docs/threat-detection-marketplace/search/06.png)

Використовуючи Light Search, ви можете знаходити детекції, вводячи запит природною мовою, який система інтерпретує та збагачує відомими псевдонімами та альтернативними посиланнями, дозволяючи пошуку знаходити всі релевантні варіації. Наприклад, якщо ви введете ім’я актора APT28, система автоматично розширить запит відомими псевдонімами, такими як Fancy Bear і Pawn Storm.

Увімкніть перемикач AI Boost, щоб посилити пошук релевантними ключовими словами від Open AI's GPT. Зверніть увагу, що при використанні AI Boost ваш пошук надсилається на зовнішній сервіс, тому не вводьте конфіденційні дані.

## Repository Selection

---

Ви можете шукати детекції в трьох доступних репозиторіях. Виберіть відповідну кнопку під рядком пошуку. За замовчуванням вибрано опцію Platform Repos.

![Знімок екрана продукту](/img/docs/threat-detection-marketplace/search/07.png)

- SOC Prime: контент, створений внутрішньою командою SOC Prime
- Threat Bounty: контент, створений розробниками з програми Threat Bounty
- SigmaHQ: контент з open-source проєкту SigmaHQ на GitHub
- Microsoft Sentinel: контент для Microsoft Sentinel з проєкту Azure-Sentinel на GitHub

За замовчуванням відображається контент з усіх репозиторіїв обраного типу. Якщо ви хочете бачити та шукати контент лише з певних репозиторіїв, виберіть їх у випадаючому списку та натисніть Apply.

![Знімок екрана продукту](/img/docs/threat-detection-marketplace/search/08.png)

Ви можете шукати та фільтрувати контент у кастомних репозиторіях так само, як контент, опублікований на TDM. Крім того, фільтрація працюватиме лише доти, доки відповідні метадані присутні в кастомному контенті.

## Search Profile

---

Виберіть Search Profile з випадаючого меню, щоб миттєво отримати результати, що відповідають інфраструктурі вашої компанії, фактичним джерелам логів і релевантним параметрам MITRE ATT&CK.

Якщо профіль позначено як Default on Search на сторінці Search Profiles, при відкритті Advanced Search цей профіль буде автоматично застосовано з налаштованими параметрами.

Якщо ви не бажаєте використовувати жодний Search Profile, виберіть опцію None. Якщо у вас ще немає налаштованих Search Profiles, за замовчуванням застосовується None. Виберіть Create New Profile у випадаючому меню, щоб створити новий Search Profile, або натисніть значок олівця, щоб редагувати налаштування профілю. Докладніше див. у розділі [Search Profiles](https://help.socprime.com/en/articles/6261210-search-profiles).

![Знімок екрана продукту](/img/docs/threat-detection-marketplace/search/09.png)

## Sorting

---

За замовчуванням застосовується сортування Recommended. Правила Sigma ранжуються на основі наших рекомендаційних алгоритмів, що підвищують найактуальніший і якісний контент, релевантний іншим експертам з кібербезпеки.

Ви можете застосувати інші варіанти сортування:

- Microsoft: контент Microsoft Sentinel, ранжований на основі наших рекомендаційних алгоритмів, що підвищують найактуальніший і якісний контент, релевантний іншим експертам з кібербезпеки
- Google SecOps: контент Google SecOps, ранжований на основі наших рекомендаційних алгоритмів, що підвищують найактуальніший і якісний контент, релевантний іншим експертам з кібербезпеки
- Top-Rated Alerts: найнадійніші поведінкові Alerts і Queries для виявлення найсерйознішої шкідливої активності
- Name: контент, відсортований за назвою
- Type: контент, відсортований за типом (YARA Rules, Snort rules, Content Pack, Query, Alert, Red tests, Premium App, Playbook, Config)
- Views: контент, відсортований за кількістю переглядів
- Downloads: контент, відсортований за кількістю завантажень
- Released: контент, відсортований за датою випуску
- Updated: контент, відсортований за датою оновлення
- Severity: контент, відсортований за серйозністю

Крім того, ви можете перемикатися між спадним (найрелевантніші спочатку) та зростаючим (найменш релевантні спочатку) порядком, натиснувши зелену стрілку біля назви обраного варіанту сортування.

## Filter Panel

---

За допомогою фільтрів ви можете звузити результати пошуку, щоб бачити лише те, що релевантно для вашої організації.

За замовчуванням Filter Panel розгорнуто. Щоб приховати його, натисніть значок Hide Filters.

![Знімок екрана продукту](/img/docs/threat-detection-marketplace/search/11.png)

Щоб знову показати панель, натисніть значок Show Filters.

![Знімок екрана продукту](/img/docs/threat-detection-marketplace/search/12.png)

Щоб додати більше фільтрів, натисніть More Expert Filters у нижньому лівому куті та встановіть прапорець біля фільтра, який ви хочете використовувати.

![Знімок екрана продукту](/img/docs/threat-detection-marketplace/search/13.png)

Обраний фільтр з’явиться на Filter Panel. Щоб видалити фільтр з панелі, натисніть More Expert Filters і зніміть прапорець біля фільтра.

### Save Filters As Search Profile

---

Коли ви вибираєте фільтри для пошуку детекцій, ви можете зберегти обраний набір фільтрів як Search Profile. Це дозволяє повторно використовувати ті самі фільтри в майбутніх пошуках, застосовуючи збережений Search Profile.

Щоб зберегти фільтри як Search Profile:

Щоб видалити всі обрані фільтри, виберіть Clear. Щоб видалити конкретний фільтр, натисніть значок хрестика біля цього фільтра.

![Знімок екрана продукту](/img/docs/threat-detection-marketplace/search/14.png)

У модальному вікні вкажіть Profile Name і натисніть Save.

![Знімок екрана продукту](/img/docs/threat-detection-marketplace/search/15.png)

Примітка: Фільтр Content Availability застосовується лише до поточного пошуку і не зберігається як Search Profile.

Після збереження фільтрів як Search Profile ви можете вибрати цей профіль з випадаючого списку Search Profile зліва. Щоб редагувати профіль, натисніть значок олівця у випадаючому списку.

Щоб дізнатися більше про Search Profile, перейдіть до [цієї статті](https://help.socprime.com/en/articles/6261210-search-profiles).

### Filters

---

### Role and Platform

Увімкніть цей фільтр, щоб шукати контент на основі вашої професійної ролі та рішення безпеки, вказаних у налаштуваннях облікового запису або під час onboarding.

Натисніть значок Gear, щоб редагувати налаштування Role and Platform. Докладніше про налаштування цього фільтра див. у розділі [Role and Platform](https://help.socprime.com/en/articles/6259871-role-and-platform).

### Content Availability

Застосуйте цей фільтр, щоб бачити контент із певним статусом доступності. Примітка: цей фільтр не релевантний для планів Old Model Enterprise.

- Free Access: правила Sigma, доступні за безкоштовними планами підписки, наприклад правила Sigma з open-source репозиторію SigmaHQ на GitHub або правила Sigma, розроблені разом з Corelight
- Promo: правило Sigma, доступне вам безкоштовно в рамках промоакції
- Unlocked: правила Sigma, розблоковані балансом Premium Sigma rule вашої команди

- Premium: правила Sigma, які ви можете розблокувати балансом Premium вашої команди. Користувачі з планом Enterprise нової моделі отримують попередньо визначений баланс у рамках підписки
- Wait to Unlock: щойно випущені premium-правила Sigma, які стануть доступними для розблокування після очікування (залежить від вашого плану)
- Enterprise Only: Content Packs і Premium Apps

Примітка: Цей фільтр можна застосувати лише під час пошуку детекцій у Platform Repos.

### Detection Type

Фільтруйте детекції за способом їх створення:

- Human-Written Behavior Rules: Детекції, створені експертами-людьми.
- Auto-Generated IOC Queries: Усі IOC-запити, пов’язані з певною загрозою або атакою, які автоматично генеруються алгоритмами SOC Prime на основі звіту про загрозу.
- AI-Generated Rules: Усі детекції, створені AI-моделями SOC Prime.

Примітка: Цей фільтр можна застосувати лише під час пошуку детекцій у Platform Repos.

### Platform

Використовуйте цей фільтр, щоб бачити лише ті правила Sigma, які мають переклади для вашої платформи безпеки або перекладені у потрібний нативний формат.

Коли вам може знадобитися цей фільтр:

- Деякі правила Sigma містять логіку детекції або необхідні джерела логів, які наразі технічно унеможливлюють генерацію перекладів у певні формати платформ безпеки.
- Правила Sigma бувають двох типів: Alerts, призначені для детекції в реальному часі (рідко генерують false positives), і Queries, призначені для threat hunting (можуть бути шумними без налаштування під вашу інфраструктуру). Ми не генеруємо правила для alerts на основі правила Sigma типу Query.

### Industries

Використовуйте цей фільтр, щоб знайти популярний контент у вашій галузі. Цей фільтр показує контент, який організації в обраній галузі активно використовували протягом останнього місяця.

Примітка: Цей фільтр можна застосувати лише під час пошуку детекцій у Platform і Community Repos.

### Log Source Product

Кожне правило Sigma вимагає певних логів для запитів. Виберіть продукти, з яких ви збираєте дані логів, щоб бачити правила з відповідними вимогами до даних.

Альтернативно, досвідчені користувачі, які добре знають мову Sigma, можуть налаштувати більш загальну фільтрацію за необхідними джерелами логів, використовуючи фільтри Sigma Product, Sigma Service і Sigma Category.

### Author

Шукайте контент, створений певним автором, вказаним у полі author правила Sigma відповідно до ліцензії DRL.

### Actors, Tools, and Techniques

Майже всі правила Sigma в TDM зіставлені з фреймворком MITRE ATT&CK. Фільтруйте за конкретними акторами, інструментами, техніками та підтехніками, які ви пріоритизуєте, щоб бачити релевантні правила Sigma.

### Show Hidden

Увімкніть цей фільтр, щоб показати контент, який було приховано в результатах Search вашою командою. Після увімкнення фільтра ви також можете встановити прапорець Only hidden, щоб показати лише ті правила, які були приховані в результатах Search.

Примітка: Цей фільтр можна застосувати лише під час пошуку детекцій у Platform Repos.

### Translate Status

Цей фільтр застосовується для контенту, який пройшов через bulk translation flow, і показує результати автоматичних масових перекладів контенту в кастомному репозиторії.

### Cloud

Фільтруйте контент для виявлення активності противника в межах певної хмарної платформи.

### Content Type

Виберіть тип контенту, який ви хочете шукати:

- Content Pack: SIEM-native ad-hoc набір SOC-контенту, скриптів, списків, панелей та інших елементів, призначених для вирішення конкретного завдання.
- Query: правило Sigma та всі його переклади, призначені для threat hunting (можуть генерувати false positives).
- Alert: правило Sigma та його переклади, призначені для детекції в реальному часі (рідко генерують false positives).
- Premium App: обмежена кількість кастомних додатків, створених командою SOC Prime для конкретних use cases.

> **Note:**
- Правила Sigma бувають двох типів: Queries і Alerts. Вони доступні за будь-яким планом підписки. Усі інші типи контенту доступні лише за планом Enterprise.
- Цей фільтр можна застосувати лише під час пошуку детекцій у Platform Repos.

### Content Action State

Фільтруйте контент за діями, які ваша організація раніше виконала над ним. Наразі доступні чотири стани дій:

- Viewed
- Downloaded
- Downloaded via API
- Deployed (якщо ви завантажили та встановили правило вручну, цей стан відображається лише якщо ви також позначили правило як deployed за допомогою значка галочки на вкладці Code)

Використовуйте перемикач Include/Exclude, щоб показати або відфільтрувати потрібний стан.

### Excluded in Scans

Увімкніть цей фільтр, щоб показати лише контент, виключений зі сканувань Attack detective.

### Severity

Фільтр пошуку за Severity описує критичність спрацьованого правила детекції. Доступні такі варіанти Severity:

- Critical: Високорелевантна подія, що вказує на інцидент. Події Critical слід перевіряти негайно
- High: Релевантна подія, яка має спричинити внутрішнє сповіщення і потребує швидкої перевірки.
- Medium: Релевантна подія, яку слід перевіряти вручну частіше.
- Low: Помітна подія, але рідко інцидент. Події з низькою серйозністю можуть бути релевантними у великій кількості або в поєднанні з іншими. Негайна реакція не обов’язкова, але рекомендується регулярна перевірка.
- Informational: Правило призначене для збагачення подій, наприклад, тегуванням. Такі правила не повинні спричиняти alerts, оскільки очікується велика кількість подій, що їм відповідають.

### Sigma Type

Правила Sigma належать до однієї з таких трьох категорій:

- IOC Sigma: IOC-правила для виявлення вразливостей або атак, які вже відбулися.
- Threat Hunting Sigma: поведінкові правила для threat hunting і проактивного кіберзахисту.
- Compliance: правила для виявлення невідповідностей стандартам compliance.

### Sigma Status

Фільтруйте правила Sigma за їх статусом:

- Stable: правила, які вважаються стабільними і можуть використовуватися в production-системах або панелях для детекції в реальному часі.
- Test/Testing: майже стабільні правила, які можуть потребувати деякого точного налаштування.
- Experimental: правила, які часто можуть давати false-positive результати, але все ще можуть ідентифікувати релевантні події.

### Sigma Product

Ідентифікатор джерела логів Sigma для вибору всіх виходів логів певного продукту, наприклад, Linux, Windows, Cisco тощо. Докладніше в [Sigma specification](https://github.com/SigmaHQ/sigma-specification/blob/main/Sigma_specification.md#log-source).

### Sigma Service

Ідентифікатор джерела логів Sigma для вибору лише підмножини логів продукту, наприклад, sysmon, ldapd, dhcp. Докладніше в [Sigma specification](https://github.com/SigmaHQ/sigma-specification/blob/main/Sigma_specification.md#log-source).

### Sigma Category

Ідентифікатор джерела логів Sigma для вибору всіх файлів логів, записаних певною групою продуктів, наприклад, process_creation, file_access тощо. Докладніше в [Sigma specification](https://github.com/SigmaHQ/sigma-specification/blob/main/Sigma_specification.md#log-source).

### Tactics

Виберіть одну або кілька [тактик MITRE ATT&CK](https://attack.mitre.org/tactics/enterprise/), щоб відфільтрувати контент, що їх охоплює.

### Data Sources

Виберіть одне або кілька [джерел даних MITRE ATT&CK](https://attack.mitre.org/datasources/), щоб відфільтрувати контент за елементами на основі відповідних логів, таких як логи PowerShell, логи автентифікації тощо.

### Data Components

Виберіть один або кілька [компонентів даних MITRE ATT&CK](https://attack.mitre.org/datasources/), щоб відфільтрувати контент за елементами на основі цих логів, таких як Application Log Content, Certificate Registration тощо.

### CVE ID

Виберіть один або кілька CVE ID, з якими пов’язані правила.

### OS

Фільтруйте контент для виявлення активності противника в межах певної платформи.

## Premium Sigma Rule Balance

---

Threat intelligence і метадані для правил Sigma разом з усіма їх перекладами завжди доступні, щоб допомогти вам у дослідженні. Код деяких правил Sigma також доступний безкоштовно. Такі безкоштовні правила позначені такими мітками:

- Free Access. Правила Sigma, доступні за безкоштовними планами підписки, наприклад правила Sigma з open-source репозиторію SigmaHQ на GitHub або правила Sigma, розроблені разом з Corelight.
- Promo. Правила Sigma, відкриті SOC Prime в рамках промоакції.

Однак більшість правил пропонуються як Premium. Щоб отримати доступ до коду Premium Sigma rule, потрібно розблокувати його балансом Premium Sigma rule.

![Знімок екрана продукту](/img/docs/threat-detection-marketplace/search/34.png)

Усі користувачі, що належать до однієї організації, мають спільний баланс Premium Sigma rule і доступ до розблокованих правил. Коли один користувач з організації розблоковує правило, воно стає доступним усім іншим.

За планами Enterprise попередньо визначений баланс Premium Sigma rules включено в підписку.

Зверніть увагу, що коли правило Sigma випускається, ви можете розблокувати його лише після певного часу очікування:

- Enterprise: Instant
- Community: 3 days

Поки недоступне, правило Sigma має статус Wait to Unlock.

![Знімок екрана продукту](/img/docs/threat-detection-marketplace/search/35.png)

## Unlocking Sigma Rules

---

Баланс Premium Sigma rule зменшується на один при кожному розблокуванні Premium Sigma rule.

Щоб розблокувати Premium Sigma rule, відкрийте вкладку Code на сторінці правила. Коли ви розблоковуєте правило вперше, з’являється запит на підтвердження. Після першого разу правила розблоковуватимуться автоматично без додаткового підтвердження.

![Знімок екрана продукту](/img/docs/threat-detection-marketplace/search/36.png)

Коли правило розблоковано, під лічильником балансу з’являється спливаюче вікно.

![Знімок екрана продукту](/img/docs/threat-detection-marketplace/search/37.png)

Після розблокування ви миттєво отримуєте повний доступ до правила Sigma та всіх його перекладів. Статус доступу правила змінюється на Unlocked.

![Знімок екрана продукту](/img/docs/threat-detection-marketplace/search/38.png)

> **Note:**
- Правила, розблоковані зі сторінки правила, стають доступними в TDM, а також в Uncoder AI.
- За планами попередньої моделі підписки деякі механізми доступності контенту (наприклад, час очікування або баланс Sigma rule) можуть відрізнятися.
- Premium Sigma rules також можна розблокувати через Dynamic Lists в CCM і через API.
- За підпискою рівня Enterprise можливість розблоковувати правила для певного користувача може бути [надана або відкликана](https://help.socprime.com/en/articles/7731214-permission-control) роллю Manager команди.

Ви можете перевірити всі правила, які ви розблокували, використовуючи опцію Unlocked фільтра Content Availability.

### Mark as Deployed

---

Після розгортання правил ви можете позначити їх як deployed (змінити їх Action State на Deployed). Для цього:

![Знімок екрана продукту](/img/docs/threat-detection-marketplace/search/50.png)

Зверніть увагу, що ця масова дія застосовується лише до тих обраних елементів контенту, до яких вона застосовна. Логіка така:

- Якщо принаймні один обраний елемент контенту не має перекладів, позначених як deployed, кнопка має напис Mark as Deployed. Після натискання та підтвердження вибору перекладів обрані переклади позначаються як deployed для всіх обраних елементів контенту. - Якщо всі обрані елементи контенту мають принаймні один переклад, позначений як deployed, кнопка має напис Mark as Undeployed. Після натискання всі переклади всіх обраних елементів контенту позначаються як undeployed.

Ви можете перекласти кілька правил одночасно за допомогою опції Translate To. Ця опція застосовується лише для контенту з Custom Repositories.

![Знімок екрана продукту](/img/docs/threat-detection-marketplace/search/51.png)

У модальному вікні виберіть мову джерела правил на панелі вводу та цільову мову для перекладу на панелі виводу.

![Знімок екрана продукту](/img/docs/threat-detection-marketplace/search/52.png)

Виберіть Translate.

Модальне вікно Translation Results підсумовує результат перекладу, показуючи відсоток контенту, який було успішно перекладено, частково перекладено або не вдалося перекласти. Виберіть View all, щоб переглянути детальну інформацію за статусами перекладу.

![Знімок екрана продукту](/img/docs/threat-detection-marketplace/search/53.png)

Коли правила успішно перекладено, на плитці правила з’являється мітка Translated. Наведіть курсор на мітку та виберіть Go to Uncoder, щоб переглянути перекладене правило в Uncoder AI.

![Знімок екрана продукту](/img/docs/threat-detection-marketplace/search/54.png)

최신 SOC Prime 릴리스에서 사용성을 개선하고 탐지 엔지니어링 워크플로를 더 직관적이고 효율적으로 만들기 위해 Uncoder AI 인터페이스를 리디자인했습니다. 이 업데이트는 진행 중인 리디자인의 일부이며, 향후 릴리스에서 추가 개선 및 향상이 계획되어 있습니다.

## Agentic Threat Research

Uncoder AI 리디자인의 일부로, 이전 New 모드의 이름이 Agentic Threat Research 모드로 변경되었으며, 위협 분석 및 연구를 위한 향상된 AI 지원 작업 공간을 제공합니다.

![제품 스크린샷](/img/docs/release-notes/v6.2.0/01.png)

Agentic Threat Research를 사용하면 사용자가 채팅과 프로젝트를 만들고 필요에 따라 관리하며, 채팅을 프로젝트로 그룹화하여 공유 컨텍스트를 정의할 수 있습니다. 프로젝트 내에서 사용자는 해당 프로젝트의 모든 채팅에 자동으로 적용되는 지침을 제공할 수 있습니다. 이를 통해 일관된 AI 응답을 보장하고 동일한 정보를 반복 입력할 필요가 줄어듭니다.

관련 대화 전반에 공유 컨텍스트를 정의하려면 사용자가 다음 단계를 따르야 합니다.

![제품 스크린샷](/img/docs/release-notes/v6.2.0/02.png)

Set Instructions – 각 프로젝트 내에서 전용 필드에 Instructions를 정의합니다.

![제품 스크린샷](/img/docs/release-notes/v6.2.0/03.png)

Start a chat – 프로젝트에서 채팅에 직접 프롬프트를 입력합니다. 동일한 프로젝트 내에서 생성되는 이후 모든 채팅도 제공된 지침을 자동으로 상속하고 적용합니다.

![제품 스크린샷](/img/docs/release-notes/v6.2.0/04.png)

사용자는 사용자 지정 프롬프트를 위해 채팅을 통해 입력하거나, Code Editor에 위협 보고서, 탐지 규칙/쿼리 또는 기타 긴 입력을 붙여넣을 수 있습니다.

![제품 스크린샷](/img/docs/release-notes/v6.2.0/05.png)

Clear Editor 버튼을 클릭하면 Code Editor의 모든 콘텐츠가 한 번에 제거됩니다.

![제품 스크린샷](/img/docs/release-notes/v6.2.0/06.png)

탐색 효율을 높이기 위해 사용자가 채팅과 프로젝트를 빠르게 찾을 수 있는 검색 바를 도입했습니다.

![제품 스크린샷](/img/docs/release-notes/v6.2.0/07.png)

또한 AI 도구(예: Attack Flow, Short/Full Summary, Decision Tree 등)를 세 가지 카테고리로 재구성했습니다.

- Generate
- Discover
- Analyze

이제 사용자는 해당 카테고리 중 하나를 선택한 다음 그 안에서 관련 도구를 선택하여 AI 도구에 액세스할 수 있습니다.

![제품 스크린샷](/img/docs/release-notes/v6.2.0/08.png)

## New Agentic AI Tool

Uncoder AI의 기존 AI 도구 외에도 단일 요청으로 완전한 위협 인텔리전스를 제공하는 새로운 종합 AI 기반 분석 도구인 Deep Threat Research를 도입했습니다. 사용자는 위협 보고서 텍스트를 붙여넣으면 다음을 포함하는 종합적인 위협 인텔리전스를 받을 수 있습니다.

- Summary – 간결한 위협 개요
- Investigation – 위협 조사를 위한 권장 조치
- Mitigation – 모범 보안 관행에 따른 위협 완화를 위한 권장 조치
- Response – 위협 위험을 최소화하기 위한 권장 대응 조치
- Actors – 위협과 관련된 위협 행위자
- Sub-techniques – 이 위협 맥락에서 적대자가 사용하는 MITRE ATT&CK 기법
- Detection Level – 위협과 연관된 탐지 유형
- Attack Flow – 적대자 활동 시각화
- Found on TDM – SOC Prime Platform의 관련 탐지
- Generated – 관련 AI 생성 탐지
- Simulation – 악성 활동 시뮬레이션

![제품 스크린샷](/img/docs/release-notes/v6.2.0/09.png)

이 AI 도구를 사용하려면 사용자가 아래 단계를 따르야 합니다.

![제품 스크린샷](/img/docs/release-notes/v6.2.0/10.png)

## Translation

Uncoder AI 리디자인의 일부로, 이전 Classic 모드의 이름이 Translation으로 변경되었습니다.

또한 사용성을 개선하기 위해 IOC Query 옵션이 Generate 탭에서 Translation 탭의 드롭다운 메뉴로 이동하여, 사용자가 텍스트나 파일의 IOC를 바로 사용할 수 있는 쿼리로 변환할 수 있습니다.

## History for Custom Content

이 SOC Prime 릴리스에서 사용자 지정 콘텐츠의 기록을 모니터링하고 추적하는 기능을 도입했습니다. Detection Rule 페이지에 새 History 탭이 추가되어 작업을 수행한 사용자, 규칙에 수행된 작업 유형(생성, 편집, 배포, 다운로드, Content List에 추가 또는 제거 등), 타임스탬프 및 영향을 받은 플랫폼 목록을 확인할 수 있습니다.

![제품 스크린샷](/img/docs/release-notes/v6.2.0/11.png)

## Threat of the Month Block Update

최신 릴리스에서 Active Threats의 Threat of the Month 블록을 향상하여 더 자세한 위협 정보를 제공함으로써 사용자가 위협을 빠르게 이해하고 관련 콘텐츠로 이동하기 쉽게 했습니다. 업데이트된 블록에는 AI Summary, Attack Flow, Detections, Simulations가 포함되며, 각 섹션에서 사용 가능한 항목 수를 보여주는 카운터도 함께 표시됩니다. 블록 내 섹션을 클릭하면 Threat of the Month 세부 정보 페이지의 해당 영역으로 바로 이동합니다.

![제품 스크린샷](/img/docs/release-notes/v6.2.0/12.png)

## New Platform Header

이 릴리스에서 모든 페이지의 플랫폼 헤더를 재구축하여 SOC Prime 제품 목록이 포함된 표준화된 드롭다운 메뉴를 도입함으로써 탐색과 사용성을 개선했습니다.

![제품 스크린샷](/img/docs/release-notes/v6.2.0/13.png)

## Detect Flow Landing Page

플랫폼 헤더의 제품 드롭다운에서 직접 액세스할 수 있는 전용 Detect Flow 랜딩 페이지를 도입했습니다. 새 페이지는 Detect Flow 기능 개요를 제공하며, Ready for More? 연락처 모달을 통해 사용자가 정보를 제출하고 제품 제공에 대해 당사 팀과 연결할 수 있습니다.

![제품 스크린샷](/img/docs/release-notes/v6.2.0/14.png)

## WIF Support for Google SecOps

Google SecOps에 대한 WIF 지원을 Attack Detective까지 확장했습니다. 이 기능을 사용하려면 사용자는 Google SecOps용 Data Plane을 구성할 때 Attack Detective 옵션을 선택하여 Attack Detective에서 조사를 활성화해야 합니다.

![제품 스크린샷](/img/docs/release-notes/v6.2.0/15.png)

플랫폼 전반에서 더 정확한 번역을 보장하기 위해 일련의 개선 사항을 도입했습니다.

## Google SecOps

규칙 구문을 조정하여 올바른 변환을 위해 Google SecOps에서 Crowdstrike Endpoint Security로의 번역을 개선했습니다.

## Coralogix

쿼리에서 index 조건을 제거하여 Coralogix용 ECS alt 번역을 개선했습니다.

## FireEye

FireEye Rule (XML) 번역 중 사용되는 날짜 형식을 업데이트했습니다.

## LimaCharlie

Sigma 규칙에 지원되지 않는 logsource product가 포함된 경우 Sigma 규칙을 LimaCharlie로 번역할 때 표시되는 오류 메시지를 개선했습니다. 업데이트된 메시지는 현재 지원되는 logsource product를 명확하게 나열합니다.

## Microsoft Sentinel

Microsoft Sentinel에서 KQL tokenizer가 때때로 `|` 문자를 올바르게 구문 분석하지 못하여 Microsoft Sentinel에서 다른 플랫폼으로의 역번역에 영향을 미치던 문제를 수정했습니다.

## CrowdStrike Next-Gen SIEM Falcon LogScale

- 쿼리가 누락되었을 때 더 명확한 오류 메시지를 추가하여 CrowdStrike Next-Gen SIEM Falcon LogScale에서 Sigma로 번역할 때 오류 처리를 개선했으며, 예상되는 필드를 지정합니다.
- `#` 문자가 포함된 필드가 잘못 번역되던 CrowdStrike Next-Gen SIEM Falcon LogScale용 대체 CIM Config 문제를 수정했습니다.

## Other Improvements

번역이 불가능한 경우 잘못된 정보 메시지가 표시되던 문제를 수정했습니다.

---

- Search Profile을 선택하지 않은 페이지에서 콘텐츠 그래픽이 표시되지 않던 문제를 수정했습니다.
- Coverage, Content for Log Sources, Explored and Deployed Trend 섹션의 데이터를 올바르게 반영하도록 차트를 수정했습니다.
- 내보낸 Log Source Coverage CSV 파일에 빈 행 두 개가 나타나던 문제를 수정했습니다.

- CrowdStrike 쿼리의 모든 필드에 새 값이 적용되지 않던 문제를 수정했습니다.
- Hunters 쿼리에서 `like` 연산자를 사용하는 필드에 값 매핑이 적용되지 않던 문제를 수정했습니다.
- QRadar 쿼리에 사용자 지정 소스가 적용되지 않던 문제를 수정했습니다.
